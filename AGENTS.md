# AGENTS.md

本文件是当前仓库的项目级规则。后续在本仓库中执行任务时，需要优先阅读并遵守本文件。

## 规则优先级

1. 本文件 `AGENTS.md` 是当前仓库的最高优先级项目规则。
2. 如果 `README.md`、任务日志、开发记录、接口文档或数据库文档与本文件存在冲突，以本文件为准。
3. 如果用户在当前任务中给出更具体的明确要求，在不违反本文件核心范围的前提下按用户要求执行。

## 当前项目状态

当前项目只进行前端静态页面编写。

- 主要技术范围：HTML、CSS、JavaScript。
- 当前不需要后端服务。
- 当前不需要数据库。
- 当前不需要接口联调。
- 当前不需要构建工具、打包配置或部署脚本。
- C 端页面由其他人负责，本仓库当前只保留非 C 端目录。
- 不要随意引入新依赖或前端框架；如确实需要，必须先说明原因、影响和替代方案。

## 目录用途

- `ecommerce-merchant/`：电商商户端相关静态页面。
- `private-chef-merchant/`：私厨上门商户端或厨师端相关静态页面。
- `private-chef-merchant-demo/`：私厨上门商户端手机静态演示页面，面向入驻店铺、私厨团队和商家负责人。
- `takeout-merchant/`：外卖商户端手机静态演示页面，面向外卖店铺老板、店长和商家员工。
- `takeout-rider/`：外卖骑手端手机静态演示页面，面向外卖配送骑手。

`takeout-rider/` 当前为外卖骑手端纯前端静态演示目录，用于展示骑手登录注册、入驻审核、接单大厅、当前配送、取餐送达、路线查看、收入明细和个人中心流程。

## 前端静态页面要求

- 页面应优先使用语义化 HTML。
- 样式应保持清晰、可维护，避免为了小改动重构整体结构。
- 交互逻辑应保持简单，优先使用原生 JavaScript。
- 需要关注基础状态：空状态、加载态、失败态、用户操作反馈。
- 真实业务接口、数据库写入、文件落盘、CSV 合并、下载路由、真实地图接入等后端或外部服务逻辑不得写入前端静态目录。
- 不要默认创建 `customer`、`client`、`user` 等 C 端目录；如确需创建，必须由用户明确要求。

## 开州金厨设计规范

本节是当前仓库后续页面、目录和静态演示内容的统一主题规范。新增或修改页面时，默认遵守本节；如与 README 中的旧描述冲突，以本 `AGENTS.md` 为准。

### 一、整体设计风格

整体是 **轻电商 + 本地生活特卖风格**，视觉关键词为：

**白底、浅灰背景、大圆角卡片、橙红色强调、价格导向、图文商品流。**

页面不要做厚重阴影，主要依靠 **留白、圆角、浅灰分区、橙红色按钮** 来区分层级。


### 十三、页面统一规范总结

1. **主色统一使用橙红色 `#FF4B1A`**，所有选中态、价格、按钮、标签都围绕它展开。
2. **背景统一浅灰，内容区统一白色卡片**，避免页面过花。
3. **商品价格视觉权重最高**，标题次之，说明文字弱化。
4. **分类页强调效率**，顶部分类 + 左侧分类 + 右侧商品列表三层结构要清晰。
5. **详情页强调转化**，大图、价格、标题、购买按钮必须突出。
6. **圆角保持 12px 左右**，形成柔和的生活电商风格。
