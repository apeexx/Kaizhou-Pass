# AGENTS.md

本文件是当前仓库的项目级规则。后续在本仓库中执行任务时，需要优先阅读并遵守本文件。

## 规则优先级

1. 本文件 `AGENTS.md` 是当前仓库的最高优先级项目规则。
2. 如果 `README.md`、任务日志、开发记录、接口文档或数据库文档与本文件存在冲突，以本文件为准。
3. 如果用户在当前任务中给出更具体的明确要求，在不违反本文件核心范围的前提下按用户要求执行。

## 当前项目状态

当前项目只进行前端静态页面编写。

- 主要技术范围：HTML、CSS、JavaScript。
- 当前不需要后端服务。
- 当前不需要数据库。
- 当前不需要接口联调。
- 当前不需要构建工具、打包配置或部署脚本。
- C 端页面由其他人负责，本仓库当前只保留非 C 端目录。
- 不要随意引入新依赖或前端框架；如确实需要，必须先说明原因、影响和替代方案。

## 目录用途

- `ecommerce-merchant/`：电商商户端相关静态页面。
- `enterprise-services-admin/`：企业服务管理端相关静态页面。
- `platform-admin/`：平台中间商管理端相关静态页面，用于后续承载审核、客服、运营、资金等管理页面。
- `private-chef-merchant/`：私厨上门商户端或厨师端相关静态页面。
- `takeout-merchant/`：外卖商户端相关静态页面。
- `takeout-rider/`：外卖骑手端相关静态页面。

当前各业务目录仅包含 `.gitkeep` 占位文件，用于让 Git 保留空目录；不代表业务页面已完成。

## 修改前检查

开始修改前需要检查：

- 当前项目是否已初始化 Git。
- 当前分支和 `git status`。
- `AGENTS.md`。
- `README.md`。
- 与本次任务直接相关的业务目录、页面文件和静态资源。

如果缺少任务日志、接口文档、数据库文档或其他说明文件，不要编造内容；根据现有文档和代码判断，并在必要时向用户确认。

## 前端静态页面要求

- 页面应优先使用语义化 HTML。
- 样式应保持清晰、可维护，避免为了小改动重构整体结构。
- 交互逻辑应保持简单，优先使用原生 JavaScript。
- 需要关注基础状态：空状态、加载态、失败态、用户操作反馈。
- 真实业务接口、数据库写入、文件落盘、CSV 合并、下载路由等后端逻辑不得写入前端静态目录。
- 不要默认创建 `customer`、`client`、`user` 等 C 端目录；如确需创建，必须由用户明确要求。

## 文档要求

- 每次修改页面、目录结构或项目约束后，需要同步更新 `README.md` 或相关文档。
- 文档中不得写入密钥、token、cookie、真实客户数据、未脱敏截图或完整签名 URL。

## 验证要求

- 修改后需要运行项目中实际存在的验证命令。
- 当前项目没有构建脚本或测试脚本时，不要假装已运行。
- 如果修改 JavaScript 文件，优先运行：

```bash
node --check <修改过的 JS 文件>
```

- 如果新增或修改 HTML 页面，需要在浏览器中打开页面进行基础展示验证。
- 完成后需要汇报验证命令、结果和当前 `git status`。
