# takeout-merchant 规则

本目录用于“外卖商户端”手机端纯前端静态演示。

## 范围

- 只使用 HTML、CSS、原生 JavaScript。
- 不接后端、不接数据库、不接真实接口。
- 不做真实登录、鉴权、支付、结算、提现或文件上传。
- 页面数据来自静态 HTML 或 `assets/js/mock.js` 的本地演示交互。
- 页面跳转使用普通 `a` 标签 `href` 或静态表单跳转。
- 视觉主题继承仓库根目录 `AGENTS.md` 的“开州金厨设计规范”，默认使用轻电商 + 本地生活特卖风格、橙红主色、浅灰背景、白色圆角卡片和弱阴影。

## 页面定位

本目录面向外卖店铺老板、店长和商家员工，用于演示店铺入驻、订单处理、商品管理、财务结算、经营数据和店铺资料管理流程。

不要把文案写成用户点餐端，也不要写成骑手端。后续修改需要保持外卖商户端静态演示定位，不得新增后端、数据库或真实接口能力。
